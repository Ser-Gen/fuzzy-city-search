Нечёткий поиск

По заказу нужно было придумать алгоритм для поиска городов, прощающий ошибки написания.

Для определения дистанции между запросом и словами использовался алгоритм поиска расстояния Дамерау-Левенштейна с настроенными коэффициентами операций.
ru.wikipedia.org/wiki/Расстояние_Дамерау_—_Левенштейна

Полученный инструмент сортирует результат в порядке от самого похожего на запрос города к самому непохожему.
— понимает неверную английскую раскладку
	«Воронеж» найдётся по запросу «Djhjyt;»
— прощает пропущенные буквы, неверный порядок букв, неверные буквы в запросе
	«Воронеж» найдётся по запросу «Вронею» и «оранж»
— понимает транслит
	«Воронеж» найдётся по запросу «Voronezh»

Попробовать!

Не хватает
— учёта схожести звучания букв
	запрос «Варонеш» должен быть круче, чем «Вдронем» при поиске слова «Воронеж»
— учёта возможной ошибки при наборе из-за близко расположенных букв на клавиатуре
	запрос «Ыопонкж» должен быть круче, чем «Ъойонюж» при поиске слова «Воронеж»
— учёта акцидентных символов, для большей универсальности
	«Hôtel La Défense Kléber»

Для исправления этого был опробован саундекс, в который пользователем хабра был добавлен русский язык и учёт акцидентных символов.
